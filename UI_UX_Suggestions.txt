========================================
  CRITICAL ISSUES (Fix Immediately)
========================================

1. NO MOBILE NAVIGATION (HAMBURGER MENU)
   -----------------------------------------------
   Problem:  Navigation links (Home, Projects, Contact) completely disappear
             on mobile. Only the logo and "Let's Build" CTA remain. Mobile
             users have NO way to navigate to specific sections.
   Impact:   Major usability problem — mobile is likely 60%+ of traffic.
   Fix:      Add a hamburger menu icon that toggles a slide-down or
             full-screen mobile navigation menu.

2. PLACEHOLDER CONTACT LINKS
   -----------------------------------------------
   Problem:  WhatsApp link is "https://wa.me/#", LinkedIn is "#", and
             Email is "mailto:#". These are non-functional placeholders.
             If a potential client clicks them, nothing happens.
   Impact:   Destroys credibility immediately.
   Fix:      Replace with real contact information, or remove contact
             cards until real data is available.

3. "I" vs "WE" — INCONSISTENT VOICE
   -----------------------------------------------
   Problem:  The intro paragraphs use first person singular ("I have over
             14 years", "My approach"), while the rest of the site uses
             "We" ("We build", "What We Do").
   Impact:   Creates confusion about whether this is a personal portfolio
             or a company profile.
   Fix:      Unify to "We" throughout — it positions the brand as a
             team/company and sounds more professional.


========================================
  VISUAL & DESIGN IMPROVEMENTS
========================================

4. HERO SECTION — NEEDS A STRONGER VISUAL ANCHOR
   -----------------------------------------------
   Problem:  The hero has a beautiful gradient background but feels empty
             in the center-bottom area. No visual element anchors the eye.
   Fix:      Add a subtle animated element — a floating laptop/phone
             mockup, abstract code visualization, or grid pattern overlay.
             This transforms it from "text on gradient" to a memorable
             first impression.

5. NAVBAR ACTIVE STATE MISSING
   -----------------------------------------------
   Problem:  No visual indicator showing which section the user is
             currently viewing. Nav links are static.
   Fix:      Add an active state that updates on scroll (e.g., highlight
             current section link with accent color and an underline).

6. "LET'S BUILD" CTA BUTTON STYLE INCONSISTENCY
   -----------------------------------------------
   Problem:  Navbar CTA uses a white pill-shaped button, while hero CTAs
             use coral filled and bordered rectangular buttons. This
             creates visual fragmentation.
   Fix:      Standardize CTAs — use consistent border-radius (either all
             pill-shaped or all slightly rounded) and ensure primary
             action color is consistent throughout.

7. SERVICE CARDS & PRINCIPLE CARDS — TOO SIMILAR
   -----------------------------------------------
   Problem:  "What We Do" (6 cards, teal icons) and "How We Work"
             (4 cards, pink icons) have identical card styling. Visitors
             can't visually distinguish capabilities from values.
   Fix:      Differentiate the sections — for example:
             - "What We Do": Keep as cards with icons
             - "How We Work": Switch to a horizontal timeline or
               numbered steps layout

8. WEB PROJECTS GRID — 5TH PROJECT CREATES ASYMMETRY
   -----------------------------------------------
   Problem:  With 5 web projects in a 2-column grid, the last project
             ("Magical Community") sits alone on the left with empty
             space to its right. Looks unfinished.
   Fix:      Options:
             a) Make the 5th project span full width (featured layout)
             b) Add a 6th card as "Let's build your next project" CTA
             c) Use a 3-column grid for web projects on desktop

9. "VIEW LIVE" LINK — LOW CONTRAST & DISCOVERABILITY
   -----------------------------------------------
   Problem:  The teal "View Live →" link at the bottom of each web
             project card is small and easily missed. No button styling.
   Fix:      Convert to a styled button or add a background pill, arrow
             icon, or hover animation. This is the primary conversion
             action for showcasing work.

10. TEXT-HEAVY INTRODUCTION CAUSES SCROLL FATIGUE
    -----------------------------------------------
    Problem:  HomeIntro section has two dense paragraphs (~100 words each)
              that occupy 2+ full mobile screens. Most visitors won't read
              this. Studies show users spend only 5.59 seconds reading.
    Impact:   Visitors bounce before reaching the project showcase.
    Fix:      Condense to 2-3 punchy sentences per paragraph, or replace
              with a bullet-point "at a glance" format with icons.

11. FOOTER — TOO MINIMAL
    -----------------------------------------------
    Problem:  Footer is a single line: "© 2026 Inovance Technologies.
              Systems that scale." This is wasted real estate.
    Fix:      Add:
              - Quick navigation links
              - Social media icons (LinkedIn, GitHub, Twitter)
              - Email/phone contact info
              - Location (if applicable)

12. MISSING SEO METADATA
    -----------------------------------------------
    Problem:  No <meta name="description"> tag and no Open Graph (OG)
              tags in <head>. When the site is shared on LinkedIn,
              WhatsApp, or social media, it shows no preview image
              or description.
    Fix:      Add meta description and OG tags to index.html:
              - <meta name="description" content="...">
              - <meta property="og:title" content="...">
              - <meta property="og:description" content="...">
              - <meta property="og:image" content="/og-image.png">


========================================
  PERFORMANCE & POLISH
========================================

13. SCROLL REVEAL HOOK — STABILITY ISSUE
    -----------------------------------------------
    Problem:  useScrollReveal hook passes options object inline as a
              dependency to useEffect. A new object is created on every
              render, which can cause unnecessary observer re-creation.
    Fix:      Move default options to a module-level constant:
              const DEFAULT_OPTIONS = { threshold: 0.1, rootMargin: "..." };

14. NO LOADING/SKELETON STATES
    -----------------------------------------------
    Problem:  Images load lazily (good!) but there's no skeleton
              placeholder or loading shimmer while images load. On slow
              connections, users see blank gray boxes.
    Fix:      Add CSS skeleton animations to .imageContainer and
              .phoneFrame backgrounds.

15. NO 404 / ERROR HANDLING
    -----------------------------------------------
    Problem:  Single-page app with hash navigation (#home, #projects,
              #contact). If someone navigates to a wrong URL, there's
              no feedback.
    Fix:      Add basic error handling or a fallback view.
